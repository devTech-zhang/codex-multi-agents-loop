# 后端技术方案

后端工程师 Agent 输出 Node.js 服务端设计方案。默认技术方向为 Node.js；框架可根据需求复杂度选择 Express、Fastify、NestJS 或轻量内置方案，但必须说明选择理由。

必须包含：

- 工程结构：代码写入 `source-code/backend`，说明目录、入口、配置、路由、服务、数据访问、测试文件位置。
- 领域模型：实体、字段、关系、状态机和约束。
- API 设计：路径、方法、请求、响应、错误码、幂等、分页、排序、过滤。
- 数据存储：本地文件、SQLite、Postgres 或内存方案的选择理由，以及迁移/初始化方式。
- 权限与安全：鉴权、输入校验、敏感信息、跨域、日志脱敏。
- 可观测性：日志、错误追踪、关键业务事件。
- 测试策略：单元测试、接口测试、异常测试、启动健康检查。
- 准出标准：服务必须可启动，核心接口必须真实请求通过，不能留下运行失败或接口报错。

方案必须足够让开发 runner 生成确定性的工程任务。
