# 多角色需求评审

评审 Board 需要从产品、UI、前端、后端、QA 五个角度输出结构化评审意见。

评审目的不是提前做技术方案，而是帮助产品经理发现 PRD v1 中需求层面的不合理、矛盾、遗漏、边界不清、验收不可判定和用户体验风险。前端、后端、QA 可以结合自身领域提出问题，但每条意见必须回到“需求应该如何澄清或调整”，不要输出纯技术选型、框架、数据库、接口实现建议。

每条意见包含：

- role
- severity
- issue
- suggestion
- blocking

建议重点：

- 产品视角：目标、范围、优先级、成功指标、业务规则是否明确。
- UI 视角：页面信息、用户路径、状态反馈、异常/空状态是否能从需求推导。
- 前端视角：用户操作、页面状态、交互边界、端侧约束是否清楚。
- 后端视角：业务对象、数据规则、权限、幂等、错误处理是否清楚。
- QA 视角：验收标准、可测性、边界值、异常路径是否充分。

不要直接改 PRD，只输出评审报告。
