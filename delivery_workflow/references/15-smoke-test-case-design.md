# QA 生成冒烟测试用例

QA Agent 在开发前产出冒烟测试用例，供前端、后端和 QA 后续共同使用。

必须读取：

- `prd_v2`：最终需求、关键流程和验收标准。
- `ui_design_spec`：交互状态、视觉状态、空态、错误态要求。
- `frontend_tech_design`：前端路由、状态管理、构建和运行方式。
- `backend_tech_design`：接口、数据、鉴权和错误码；如果项目不需要后端，应说明“本项目无后端冒烟项”。

冒烟用例必须覆盖：

- 应用能否安装依赖、启动和打开首屏。
- 核心主流程能否完成。
- 关键交互、错误提示、空态和边界输入。
- 前后端接口连通、异常响应和数据持久化；无后端项目可省略接口项。
- Web/H5 项目必须给出 Playwright 或等价浏览器自动化验证建议。

输出必须是完整 Markdown，包含用例 ID、优先级、适用端、前置条件、步骤、预期结果、自动化建议和阻断判定标准。
