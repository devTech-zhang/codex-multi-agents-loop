# UI 设计规范

UI 设计师 Agent 输出完整 UI 设计规范，为前端实现提供可执行的视觉、布局、组件和交互依据。

必须包含：

- 产品定位与设计原则：说明目标用户、使用场景、信息密度、情绪基调和效率优先级。
- 信息架构：页面层级、导航结构、核心对象、用户路径。
- 页面清单：每个页面的目标、主要内容、入口、出口、空状态、加载态、异常态、权限态。
- 视觉主题与氛围：参考 DESIGN.md 写法，明确整体视觉语言，不要只写“简洁现代”。
- 颜色角色：语义色、品牌色、中性色、背景/前景/边框/状态色，必要时给出 hex 和使用边界。
- 字体规则：字号层级、字重、行高、数字/表格/按钮文字规则。
- 组件规范：按钮、输入、表单、列表、卡片、弹窗、导航、反馈组件的尺寸、状态和禁用规则。
- 布局原则：栅格、间距、触控热区、移动端/桌面端断点、滚动与固定区域。
- 交互规则：点击、提交、撤销、错误提示、成功反馈、二次确认、键盘/触屏行为。
- Do / Don't：列出前端实现必须遵守的设计护栏和反例。
- Agent Prompt Guide：给前端 Agent 的简短执行提示，帮助其按本规范实现。

设计规范应参考 [getdesign.md](https://getdesign.md) 中 DESIGN.md 的结构：视觉主题、颜色角色、字体层级、组件样式、布局原则、深度层级、Do/Don't、响应式行为和 Agent 提示指南。输出必须足够让前端完成高质量实现。
