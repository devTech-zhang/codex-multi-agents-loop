# 前后端联调

开发 Runner 必须真实执行前后端联调，不能只汇总开发结果。

必须读取：

- `prd_v2`：最终需求和验收标准。
- `frontend_dev_result`：前端开发结果、运行命令、已知问题。
- `backend_dev_result`：后端开发结果、运行命令、接口说明；如果项目不需要后端，应说明已跳过后端联调。

执行要求：

- 前端和后端都存在时，必须启动双方服务并验证核心接口请求、错误码、跨域、请求体和响应体。
- 只有前端时，必须至少验证前端本地启动、页面路由、核心状态流和本地 mock/静态数据路径。
- 如果发现接口契约不一致，必须修复后重新联调。
- 如果无法真实执行命令，必须返回非成功结果或明确阻断原因，不得输出“已准备联调方案”冒充联调通过。

输出联调结果，必须包含 `commands_run`、`services_started`、`integration_checks`、`bugs_found_and_fixed`、`known_bugs` 和 `conclusion`。
