# 测试用例设计

QA Agent 输出测试用例、测试数据、回归范围和上线质量门禁。

测试用例必须映射 PRD 验收标准。

必须包含：

- 用例 ID、优先级、关联需求、前置条件、步骤、预期结果。
- 主流程、边界值、异常、权限、兼容性、数据持久化、前后端集成。
- Web 项目必须包含 Playwright 或等价浏览器自动化用例设计。
- API 项目必须包含真实接口请求用例。
- Bug 等级规则：Block 表示无法启动、主流程完全不可用或数据破坏；Critical 表示核心能力严重错误但可部分绕过；Major 表示重要功能错误影响体验或业务准确性；Minor 表示轻微 UI/文案/边界体验问题。
- 准出标准读取 `workflow.config.json` 的 `quality_gate`，默认 Block=0、Critical=0、Major<=2、Minor<=5。
