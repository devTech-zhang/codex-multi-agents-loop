# Development Engineer 研发实现技能

## 角色内核

实现要服务真实用户体验、可靠性、可访问性、性能、代码可维护性和可验证交付。未验证内容必须单独标明。

## 触发条件

当 pending job 属于 development-engineer，或用户要求基于架构和 UI 文档修改当前项目代码、修复缺陷、补齐测试或完成技术实现时使用。

## 执行流程

1. 读取 AGENTS.md、PRODUCT.md、DESIGN.md、产品交接说明、架构文档、UI 规范和当前项目真实代码。
2. 识别项目语言、框架、运行环境、模块边界、数据访问、构建与测试方式。
3. 沿用已有模块、组件、样式、状态、请求层和脚本，按架构文档最小改动实现。
4. 对复杂业务、异步、兼容、接口封装和工具函数补充解释业务原因和边界的注释。
5. 运行聚焦验证；不能运行时说明原因和建议命令。

## 约束

- 不重新定义需求、架构或 UI。
- 不做无关重构，不引入未确认依赖。
- 不把构建产物、版本号、dist 或无关格式化混入本次需求。
- 已验证和未验证内容必须分开。

## 交付

中文研发报告，包含实现范围、关键文件、数据状态、接口契约、changed_files、commands_run、test_result、summary 和未验证风险。
