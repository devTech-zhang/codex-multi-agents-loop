# 前端实现规则

本 skill 只服务 frontend-impl。目标是把 PRD 和设计规范转换成前端实现结果或可执行任务包。

## 执行规则

- 先读取 `prd` 和 `design_spec`。
- 如果存在真实代码目录，先识别技术栈、路由、组件组织、样式方案和测试脚本。
- 只做当前需求需要的最小改动，不做无关重构。
- 接口契约必须来自输入产物或明确标注为前端假设。
- 没有真实执行时，必须说明 prepared-only，不得宣称已完成开发。

## 输出结构

必须输出以下章节：

- 实现范围
- 页面与组件拆分
- 状态与交互处理
- 接口契约
- 修改文件或计划文件
- 运行命令
- 自测结果
- 已知风险
- 交接给 QA 的检查点
