# 后端实现规则

本 skill 只服务 backend-impl。目标是把 PRD 转换成后端实现结果或可执行任务包。

## 执行规则

- 先读取 `prd`，抽取接口、数据、权限、边界和验收标准。
- 如果存在真实代码目录，先识别服务框架、路由层、数据访问层、配置方式和测试脚本。
- 不引入未确认的数据库、缓存、队列或第三方服务。
- 接口必须写清请求、响应、错误、权限和幂等规则。
- 没有真实执行时，必须说明 prepared-only，不得宣称已完成开发。

## 输出结构

必须输出以下章节：

- 实现范围
- 接口清单
- 数据模型
- 权限规则
- 错误码
- 日志与监控点
- 修改文件或计划文件
- 运行命令
- 自测结果
- 已知风险
- 交接给前端和 QA 的检查点
