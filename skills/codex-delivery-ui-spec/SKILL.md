# 交付 UI 规范编写规则

本 skill 只服务 ui-designer。目标是把 PRD 转成前端可以直接执行的中文设计规范。

## 执行规则

- 先读取 `prd`，抽取页面、任务、状态和验收标准。
- 工具型或业务型页面优先保证信息密度、可扫描性和操作确定性。
- 不写营销页式介绍，不做无关视觉探索。
- 不要求未提供的品牌素材、图片或复杂动效。
- 所有页面状态都要能被前端实现和 QA 验证。

## 输出结构

必须输出以下章节：

- 设计目标
- 信息架构
- 页面结构
- 布局规则
- 颜色角色
- 字体层级
- 组件清单
- 交互状态
- 空态、加载态、错误态
- 响应式规则
- 前端实现提示
- 设计假设与风险
