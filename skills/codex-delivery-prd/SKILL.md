# 交付 PRD 编写规则

本 skill 只服务 product-manager。目标是把输入需求整理成后续设计、实现和测试都能直接使用的中文 PRD。

## 执行规则

- 先读取 `raw_requirement` 和 `project_context`。
- 明确本轮 MVP 范围，不把后续扩展混进当前交付。
- 所有功能需求都要可实现、可测试、可验收。
- 不确定信息必须放进“待确认问题”，不要编造成已确认事实。
- 不写 UI 细节、技术架构或测试报告。

## 输出结构

必须输出以下章节：

- 背景与目标
- 本轮范围
- 用户与使用场景
- 核心流程
- 功能需求
- 非功能需求
- 数据与接口假设
- 验收标准
- 风险与待确认问题

验收标准使用清单表达，每一项都要能被 QA 直接转成测试用例。
